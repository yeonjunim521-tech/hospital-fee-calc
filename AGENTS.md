# AGENTS.md

## 프로젝트 기본 응답 규칙

- 이 프로젝트에서는 기본적으로 caveman mode(짧고 압축된 응답 모드)를 항상 사용한다.
- 기본 강도는 full이다.
- 답변은 한글로 한다.
- 필수 영어 용어는 처음 등장 시 영어와 한글 뜻을 함께 쓴다.
- 불필요한 인사, 장황한 설명, 반복 설명은 줄인다.
- 단, 안전 경고, 오류 원인, test(테스트), build(빌드), deploy(배포), Git/GitHub 절차는 생략하지 않는다.
- 사용자가 `stop caveman`, `normal mode`, `일반 모드`라고 말하면 이 대화에서는 caveman mode를 끈다.
- 사용자가 다시 `caveman mode`, `caveman 모드`, `짧게`, `압축해서`라고 말하면 다시 켠다.

## 작업 관리 규칙

- 작업 목록은 프로젝트 루트의 `TASKS.md` 하나만 사용한다.
- `TASKS.md`에는 현재 작업 하나만 남긴다.
- 이전 작업은 `docs/TASK_HISTORY.md` 같은 log(기록) 파일에 보관한다.
- 상태 표기는 `[ ]` 미완료, `[/]` 진행 중, `[x]` 완료를 사용한다.

## 작업 전후 규칙

- 수정 전 실제 파일과 구조를 먼저 확인한다.
- 추측으로 파일명, 함수명, API(응용프로그램 인터페이스), 설정값을 만들지 않는다.
- 변경 전 어떤 파일을 왜 수정할지 짧게 설명한다.
- 변경 후 무엇이 바뀌었는지 비전문가도 이해할 수 있게 요약한다.
- 가능하면 실행, build(빌드), test(테스트)로 확인한다.
- 검증하지 못한 것은 "검증 못함"이라고 표시한다.

## 금지

- 기존 파일 삭제, 대량 덮어쓰기, `git reset --hard`, `git clean -fd`, force push(강제 푸시)를 하지 않는다.
- `.env`, API key(API 키), token(토큰), password(비밀번호), secret(비밀값)을 노출하지 않는다.
- 사용자가 요청하지 않은 큰 refactor(구조 변경)를 하지 않는다.
