# 작업 목록 운영 규칙

기능 추가, 수정, 검증 작업을 시작하기 전에는 이 파일에 작업 목록을 먼저 작성하거나 갱신한다.

## 진행 방식

- 작업 전 `TASKS.md`에 목표, 범위, 체크리스트를 작성한다.
- 사용자가 작업 목록을 확인한 뒤 진행한다.
- 진행 중에는 완료된 항목을 체크하거나 상태를 갱신한다.
- 작업이 끝나면 검증 결과와 남은 후속 작업을 기록한다.
- 새 작업을 추가할 때는 최근 3개 작업만 남긴다.
- 완료된 지 오래된 작업은 이 파일에서 삭제한다.
- 오래된 기록 확인이 필요하면 GitHub commit(커밋) history(기록)에서 확인한다.

## 진행 결과

- `.dev.vars` 파일은 현재 프로젝트 폴더에 존재하지 않는다.
- 관리자 토큰값이 들어간 로컬 환경 파일도 발견되지 않았다.
- `.gitignore`에 `.dev.vars`, `.env`, `.env.*` 제외 설정이 유지되어 있다.
- 이번 확인/문서 정리는 GitHub에 commit(커밋)하거나 push(푸시)하지 않았다.
