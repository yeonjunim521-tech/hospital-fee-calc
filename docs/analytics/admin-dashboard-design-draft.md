# 관리자 검색 통계 대시보드 디자인 초안

## 디자인 목표

이 화면은 Google Analytics용 화면이 아니다.  
관리자가 방문자 검색 로그를 보고 "어떤 수가/키워드를 추가해야 하는지" 판단하는 내부 운영 화면이다.

핵심 질문은 세 가지다.

1. 사람들이 무엇을 많이 찾는가?
2. 어떤 검색어가 결과가 없는가?
3. 그래서 오늘 무엇을 DB에 추가하거나 키워드로 보강해야 하는가?

## 1차 화면 구성

```text
┌──────────────────────────────────────────────────────────────┐
│ 검색 통계 관리                                                │
│ 기간 선택 [7일][30일][90일]  새로고침                         │
├──────────────────────────────────────────────────────────────┤
│ 총 검색수   결과 있음   결과 없음   무결과율   클릭수          │
│ 1,240      980        260        21%       430               │
├──────────────────────────────┬───────────────────────────────┤
│ 많이 검색된 키워드            │ 결과 없는 검색어                │
│ 고압산소치료  18회            │ 뇌 ct         5회               │
│ 무통          12회            │ 항생제        2회               │
│ x ray         10회            │ 하반시마취    2회               │
├──────────────────────────────┴───────────────────────────────┤
│ 추가 후보 큐                                                   │
│ 키워드 | 추천 분류 | 연결 코드 | 우선순위 | 상태 | 메모        │
├──────────────────────────────────────────────────────────────┤
│ 검색어 상세                                                     │
│ 선택한 검색어의 일별 추이, 입력 페이지, 클릭 항목               │
└──────────────────────────────────────────────────────────────┘
```

## 상단 영역

### 제목

`검색 통계 관리`

보조 문구:
`방문자가 찾았지만 결과가 부족한 항목을 확인하고 DB 보강 후보를 정리합니다.`

### 필터

- 기간: `7일`, `30일`, `90일`, `365일`
- 결과 상태: `전체`, `결과 있음`, `결과 없음`
- 최소 검색수: `1회 이상`, `2회 이상`, `5회 이상`
- 분류: `전체`, `검사`, `시술`, `수술`, `기타`

1차 구현에서는 기존 기간 선택만 유지해도 된다.  
나머지 필터는 다음 단계에서 추가한다.

## KPI 카드

상단에는 숫자 카드 5개를 둔다.

| 카드 | 의미 | 관리자가 보는 이유 |
|---|---|---|
| 총 검색수 | 선택 기간 전체 검색 횟수 | 방문자 검색 사용량 확인 |
| 결과 있음 | 결과가 나온 검색 횟수 | 현재 DB가 대응한 검색 규모 확인 |
| 결과 없음 | 결과 0건 검색 횟수 | DB/키워드 보강 대상 확인 |
| 무결과율 | 결과 없음 / 총 검색수 | 검색 품질 상태 확인 |
| 결과 클릭수 | 검색 결과를 실제 클릭한 횟수 | 결과가 쓸모 있었는지 확인 |

## 메인 표 1: 많이 검색된 키워드

목적: 실제 수요가 많은 항목 확인.

컬럼:

- 검색어
- 검색수
- 평균 결과 수
- 클릭수
- 마지막 검색일
- 추천 조치

추천 조치 예:

- `유지`: 결과가 충분하고 클릭도 있음
- `키워드 보강`: 결과는 있지만 클릭이 낮음
- `항목 확인`: 검색수는 높은데 결과 수가 적음

## 메인 표 2: 결과 없는 검색어

목적: 없는 기능/없는 수가/없는 키워드 찾기.

컬럼:

- 검색어
- 검색수
- 마지막 검색일
- 입력 페이지
- 유사 키워드
- 추천 분류
- 추가 후보
- 메모

상태 표시:

- `높음`: 검색수 5회 이상이고 결과 없음
- `중간`: 검색수 2회 이상이고 결과 없음
- `낮음`: 1회성 검색
- `보류`: 너무 넓은 단어

예:

| 검색어 | 판단 |
|---|---|
| `고압산소치료` | 수가 항목 추가 대상 |
| `뇌 ct` | 기존 CT 항목 키워드 보강 |
| `항생제` | 너무 넓음. 약제 DB 없으면 보류 |
| `하반시마취` | 오타 키워드 alias 추가 |

## 추가 후보 큐

검색 로그를 실제 작업 목록으로 바꾸는 영역이다.

컬럼:

- 키워드
- 항목명
- 코드
- 분류
- 우선순위
- 상태
- 담당자
- 수정일
- 메모

상태:

- `검토 전`
- `키워드만 추가`
- `수가 항목 추가 필요`
- `추가 완료`
- `보류`

1차 구현에서는 담당자 없이 `상태`와 `메모`만 있어도 충분하다.

## 상세 패널

표에서 검색어 하나를 누르면 오른쪽 또는 아래에 상세 패널을 보여준다.

표시 내용:

- 선택 검색어
- 총 검색수
- 결과 없음 횟수
- 최근 검색일
- 검색한 페이지
- 클릭된 항목
- 관리자 메모

작업 버튼:

- `키워드 추가 후보로 표시`
- `수가 항목 추가 후보로 표시`
- `보류`
- `삭제`

## 첫 구현 추천 범위

너무 크게 만들지 말고 아래까지만 먼저 만든다.

1. 상단 KPI 카드
2. 많이 검색된 키워드 표
3. 결과 없는 검색어 표
4. 삭제 버튼 유지
5. 결과 없는 검색어에 `메모` 입력칸 추가

## 나중에 추가할 기능

- 검색어 자동 묶기: `뇌 ct`, `노 ct`, `뇌ct`
- HIRA 코드 후보 자동 추천
- 추가 완료 후 검색 결과 개선 여부 추적
- CSV 내보내기
- 상태별 필터

## 디자인 톤

- 내부 운영 도구라서 화려할 필요 없다.
- 숫자와 표가 먼저 보여야 한다.
- 카드 남발보다 표 중심이 좋다.
- 빨간색은 삭제/위험에만 사용한다.
- 결과 없는 검색어는 노란색 또는 주황색 계열로 표시한다.
- 추가 완료는 초록색으로 표시한다.

## 사용자 흐름

```text
관리자 접속
→ 기간 선택
→ 결과 없는 검색어 확인
→ 로컬 DB에 있는지 확인
→ 있으면 키워드 보강
→ 없으면 HIRA 수가표 확인
→ 추가 후보 큐 상태 변경
→ 다음 배포 때 반영
```
